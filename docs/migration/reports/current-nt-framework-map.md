# Current nt Framework Map

Files inspected: root `CMakeLists.txt`, `engine/CMakeLists.txt`, `engine/include/noveltea/*`, `engine/src/renderer_bgfx.cpp`, `engine/src/engine.cpp`, `apps/sandbox/main.cpp`.

Key findings:

- The current runtime is a compact SDL3 platform layer, bgfx renderer, debug UI facade, runtime UI facade, and sandbox app.
- bgfx shader headers are generated by CMake from `engine/shaders/bgfx/*.sc` into `engine/src/shaders`.
- Dear ImGui is already separated behind `NOVELTEA_ENABLE_DEVTOOLS` and should remain dev/debug only.
- Runtime UI was a RmlUi package-link scaffold without a render/system/file backend.
- The sandbox delegates entirely to `noveltea::App`, so framework demos belong in `Engine::render()` or renderer-owned demo hooks until sandbox modes exist.

Risks:

- Web/Android bgfx dependency fetching is still network-based in the existing project. This report did not change that behavior.
- RmlUi package availability is optional and target-limited.

Recommended next action: keep bootstrap additions small and feature-gated, then verify Linux/Web/Android presets.
